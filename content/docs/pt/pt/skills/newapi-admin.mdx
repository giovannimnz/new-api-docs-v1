## Escopo previsto

A ideia é permitir que administradores executem tarefas operacionais direto do editor,
sem depender do painel web para cada ajuste do dia a dia.

### Áreas cobertas

| Área | Exemplos previstos |
|------|--------------------|
| Canais | listar, testar, ativar/desativar, ajustar prioridade |
| Utilizadores | consultar saldo, alterar grupo, bloquear/desbloquear |
| Sistema | ler e alterar configurações, sincronizar ratios |
| Observabilidade | ver logs, estatísticas e saúde do ambiente |
| Tokens e redemptions | visão global, criação em lote, auditoria |

## Diferença para a Skill `newapi`

| | `newapi` | `newapi-admin` |
|---|---------|----------------|
| Público | utilizadores / developers | administradores |
| Escopo | modelos, tokens pessoais, saldo, ajuda | canais, utilizadores, config e logs |
| Permissão | token de utilizador | token administrativo |
| Estado | disponível | em desenvolvimento |
