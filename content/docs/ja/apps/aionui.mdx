## 主要機能

### 💬 マルチセッションチャット
- **マルチセッション + 独立したコンテキスト** - 複数のチャットセッションを同時に開くことができ、各セッションは独立したコンテキスト記憶を持ちます
- **ローカルストレージ** - すべての会話データはローカルのSQLiteデータベースに保存され、失われることはありません

### 🤖 マルチモデルサポート
- **マルチプラットフォームサポート** - Gemini、OpenAI、Claude、Qwenなどの主要モデルをサポートし、柔軟に切り替え可能
- **ローカルモデルサポート** - Ollama、LM Studioなどのローカルモデルデプロイメントをサポート

### 🤝 マルチエージェントモード
- **複数のAIエージェントを同時に実行** - 複数のAIエージェント（例：Gemini CLI、Claude Code、Codex、OpenCode、Qwen Code、Goose CLI、Auggieなど）を同時に実行可能
- **MCPによる一元管理** - Model Context Protocol (MCP) を通じてすべてのエージェントを一元的に管理および設定し、操作プロセスを簡素化
- **スキル設定** - 異なるエージェントに専用のSkillsを設定し、エージェントの能力を拡張
- **アシスタントのカスタマイズ** - カスタムアシスタント設定をサポートし、パーソナライズされたAIワークフローを構築
- **独立した設定** - 各エージェントは独立して設定および使用でき、互いに干渉しません
- **柔軟な切り替え** - 異なるエージェント間で柔軟に切り替え、さまざまなシナリオのニーズに対応

### 🗂️ ファイル管理
- **ファイルツリー表示 + ドラッグ＆ドロップアップロード** - フォルダーのようにファイルを閲覧し、ファイルやフォルダーのドラッグ＆ドロップによるワンクリックインポートをサポート
- **スマート整理** - AIにフォルダーの整理を依頼でき、自動分類

### 📄 プレビューパネル
- **9種類以上の形式プレビュー** - PDF、Word、Excel、PPT、コード、Markdown、画像などの形式をサポート
- **リアルタイム追跡 + 編集可能** - ファイルの変更を自動的に追跡し、Markdown、コード、HTMLのリアルタイム編集とデバッグをサポート

### 🎨 AI画像生成と編集
- **スマート画像生成** - Gemini 2.5 Flash Image Preview、Nano、Bananaなどの多様な画像生成モデルをサポート
- **画像認識と編集** - AI駆動の画像分析および編集機能

### 🌐 マルチチャネルアクセス
- **WebUIリモートアクセス** - ブラウザを通じてネットワーク上のあらゆるデバイスからアクセス可能、モバイルデバイスもサポート
- **Telegram連携** - Telegramボットを介したインタラクションをサポート
- **Feishu（飛書）連携** - Feishu（飛書）を介したアクセスとインタラクションをサポート
- **ローカルデータセキュリティ** - すべてのデータはローカルのSQLiteデータベースに保存され、サーバーデプロイメントに適しています

## NewAPI接続方法

### パラメータ入力

プロバイダータイプ：NewAPIがサポートするタイプ  
APIキー：NewAPIで取得  
APIアドレス：NewAPIサイトアドレス（例：`https://your-newapi-domain.com/v1`）

### 設定手順

1. **NewAPIでAPIキーをコピー**
   ![APIキーをコピー](/assets/aionui/copy_apikey.png)

2. **AionUi設定を開く**
   - AionUiで設定ページに移動
   - モデル設定タブを見つける
   - 「モデルを追加」をクリック
   ![設定を開く](/assets/aionui/add-model-1.png)

3. **新しいプロバイダーを追加**
   - 「モデルを追加」をクリック
   - NewAPIを選択
   ![NewAPIプロバイダーを追加](/assets/aionui/newapi_provider.png)

4. **API情報を設定**
   - APIアドレス：NewAPIサイトアドレスを入力してください（形式：`https://your-domain.com/v1`）
   - APIキー：NewAPIコンソールからコピーしたAPIキーを貼り付けます

5. **モデルを追加**
   - ドロップダウンから追加するモデルを選択
   - モデル名はNewAPIで設定されたモデル名と一致している必要があります
   - 適切なリクエストプロトコルを選択

6. **使用を開始**
   - チャットページに戻る
   - 設定済みのNewAPIモデルを選択して会話を開始
